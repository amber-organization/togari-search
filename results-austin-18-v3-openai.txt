METHODOLOGY NOTES
──────────────────────────────────────────────────────────────
This report uses three principled refinements beyond a plain cosine
similarity pipeline. Each is a modeling decision, not a tuned knob.

 1. Density-weighted text vectors.  For each pair and each text
    vector V (identity/personality/experience/interest) we multiply
    the base weight by density_V = min(words_A, words_B) / 100
    (capped at 1.0), then renormalise. A 0.6 cosine between two
    8-word blurbs is treated as a weaker signal than a 0.6 between
    two 80-word descriptions. Vectors where either side wrote
    nothing contribute zero weight automatically.

 2. Per-prediction confidence.  Alongside every score we report
    confidence = 0.50*text_completeness + 0.30*structured_coverage
                  + 0.20*graph_signal.
    Bands: >0.70 high, 0.40–0.70 medium, <0.40 low. A low-
    confidence score is not wrong — it is under-informed, and
    should be weighted accordingly when acting on it.

 3. Guest-type complementarity matrix.  complementary_type now
    uses a 3x3 lookup (storyteller/investigator/listener) grounded
    in dinner conversation dynamics: storyteller+listener = 1.00,
    investigator pairings in [0.65, 0.85], two storytellers 0.45,
    two listeners 0.35. Unknown taxonomies fall back to 0.50, so
    non-Blind8 partner data does not regress.
